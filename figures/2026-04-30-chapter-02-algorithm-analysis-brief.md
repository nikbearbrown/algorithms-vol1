# Figure brief — Chapter 02: Algorithm Analysis

## Recommended figures

### Figure 1 — Growth-rate curves
**Path:** `../images/chapter-2-growth-rate-curves.jpg`
**Description:** Log-log plot of the standard growth rates — `1`, `log n`, `n`, `n log n`, `n²`, `n³`, `2ⁿ`, `n!` — over `n` from 1 to 1,000,000. X-axis log-scale, Y-axis log-scale. Curves labeled at the right edge. Shading at `n = 10⁶` showing where polynomial bounds remain practical and exponential bounds blow past wall-clock-feasible.
**Use:** referenced in Section 3 (Big O / Omega / Theta) when listing growth rates from cheapest to most expensive.

### Figure 2 — Insertion sort vs merge sort crossover benchmark
**Path:** `../images/chapter-2-sort-crossover-benchmark.jpg`
**Description:** Wall-clock plot, X-axis `n` from 1 to 1,000 (linear), Y-axis time in microseconds. Two curves: insertion sort (steeper polynomial), merge sort (linearithmic). Crossover annotated at approximately `n = 32` to `n = 64`. Annotation: "Below crossover, insertion sort wins on cache locality and tight inner loop. Above crossover, asymptotic dominance takes over."
**Use:** referenced in §6 (Cache effects) and the worked example.
**Source data:** to be generated by the companion-page benchmark harness; figure should match.

### Figure 3 — Memory hierarchy latency
**Path:** `../images/chapter-2-memory-hierarchy.jpg`
**Description:** Bar chart, log Y-axis, showing typical latency for register / L1 / L2 / L3 / RAM / SSD / HDD on the same machine. Each bar labeled with both a representative latency number and a typical capacity range.
**Use:** referenced in §6 to anchor the cache-effects argument visually.
**Note:** caption should mark numbers as representative, with `[verify]` resolved before publication.

### Figure 4 — Master theorem decision flow
**Path:** `../images/chapter-2-master-theorem-flow.jpg`
**Description:** Decision tree starting from "T(n) = aT(n/b) + f(n)" → identify a, b, f(n) → compare f(n) to n^(log_b a) → three cases with output bounds. One branch leads to "use recursion tree instead" if regularity fails.
**Use:** referenced in §4 (Recurrences and the master theorem). Could replace the master theorem reference card or live alongside it on the companion page.
